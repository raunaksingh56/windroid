## вңЁ What is WinDroid?

**WinDroid** is an open-source Android app that turns your phone into a bootable Windows USB creator. Plug in a USB OTG drive, pick your ISO, and flash вҖ” the USB boots on any PC automatically. No PC required, no root, no BIOS changes.

| Version | Answer File | Notes |
|---------|-------------|-------|
| рҹӘҹ Windows XP | `winnt.sif` | Legacy text-format, full unattend |
| рҹӘҹ Windows 7 | `autounattend.xml` | ei.cfg injection, activation skip |
| рҹӘҹ Windows 8 / 8.1 | `autounattend.xml` | Online account bypass, SmartScreen off |
| рҹӘҹ Windows 10 | `autounattend.xml` | Telemetry & Cortana off, offline setup |
| рҹӘҹ Windows 11 | `autounattend.xml` | TPM / SecureBoot / RAM bypass, offline setup |

## рҹҶ• Changelog

### v1.1.0 вҖ” Bug Fix + Feature Release

#### рҹҗӣ Critical Fixes

| # | File | Bug | Impact |
|---|------|-----|--------|
| 1 | `IsoExtractor.kt` | `splitAndWrite` buffered full WIM/ESD chunk (up to 3.8 GB) in a `ByteArrayOutputStream` before writing | **OOM crash** on every Windows 11 flash вҖ” install.wim routinely exceeds 4вҖ“5 GB |
| 2 | `IsoExtractor.kt` | `InputStream` never closed after each file вҖ” one FD leaked per file | **FD exhaustion crash** on large ISOs (thousands of files) |
| 3 | `HybridBootManager.kt` | `dd bs=446 count=1` only wrote 446 bytes вҖ” the `0x55AA` boot signature was silently never reaching the USB | USB **never bootable** on blank/corrupted drives even after MBR write "succeeded" |
| 4 | `HybridBootManager.kt` | `lastOrNull()` on `ls /dev/block/sd*` could pick internal eMMC as "USB drive" | Risk of writing MBR directly to **internal storage** |

#### рҹҗӣ High Severity Fixes

| # | File | Bug | Impact |
|---|------|-----|--------|
| 5 | `HybridBootManager.kt` | `readMBR` called `waitFor()` after `readBytes()` вҖ” deadlock on some root environments | App **hangs forever** mid-flash |
| 6 | `HybridBootManager.kt` | `writeMBRRoot` read stderr after `waitFor()` вҖ” deadlock if stderr buffer fills | App **hangs** on slow USB writes |
| 7 | `AutounattendGenerator.kt` | `disableCortana` emitted `<DisableAutoDaylightTimeSet>` вҖ” completely wrong XML element | Cortana **never actually disabled** despite toggle being on |
| 8 | `UsbWriter.kt` | `ei.cfg` generated but never written вҖ” only `autounattend.xml` was ever flashed | Windows 7/8 **edition selection broken** entirely |

#### рҹҗӣ Medium Severity Fixes

| # | File | Bug | Impact |
|---|------|-----|--------|
| 9 | `IsoExtractor.kt` | Joliet `trimEnd(';','1',' ')` corrupted filenames ending in digits | `setup1` вҶ’ `setup`, `boot1.txt` вҶ’ `boot.txt` |
| 10 | `UsbWriter.kt` | `getOrCreateDir` called `createDocument` unconditionally вҖ” duplicated folders on re-flash | `sources/` created twice, `ei.cfg` injection fails |
| 11 | `WinDroidViewModel.kt` | `isFlashing` stayed `true` on error вҖ” progress spinner kept spinning after failure | UI **stuck** showing "Flashing..." forever on error |
| 12 | `WinDroidViewModel.kt` | `generateAll()` called on main thread before IO coroutine | Potential **ANR** when version detection is added |
| 13 | `HomeScreen.kt` | `activeTweaks` count missing 5 toggles (BypassNRO, SecureBoot, RAM, CPU, etc.) | Badge always showed **wrong number** |
| 14 | `HybridBootManager.kt` | Truncated 50-byte fallback MBR stub silently written instead of failing clearly | Legacy BIOS boot appeared to succeed but PC **never booted** |

#### вңЁ New Features

- **Windows 10 / 11 Offline Account Setup** (`BypassNRO`) вҖ” previously "Skip Microsoft Account" only hid the account UI; Setup still blocked until internet connected. Now injects `BypassNRO` + `BypassNROCheck` into `LabConfig` during WinPE вҖ” the actual network gate is removed, not just hidden. Toggle in Tweaks screen, on by default.
- **USB block device detection via `/sys/block/sdX/removable`** вҖ” replaces the dangerous `lastOrNull()` heuristic with kernel-reported removability flags.
- **Streaming WIM/ESD chunk writes** вҖ” replaced `ByteArrayOutputStream` buffering with direct 8 MB incremental USB writes. Windows 11 flashes no longer risk OOM.
- **Cortana disable via `FirstLogonCommands`** вҖ” correct registry key (`AllowCortana=0`) instead of the wrong `DisableAutoDaylightTimeSet` element.
- **CI fixed** вҖ” added `permissions: contents: write` to `build.yml` so GitHub Actions can create releases.

в”Ӯ  ISO 9660 Parser вҶ’ USB Writer вҶ’ Boot Setup вҶ’ Answer File Inject  в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ

- вң… **Full ISO 9660 + Joliet parser** вҖ” reads any Windows ISO natively, no dependencies
- вң… **Streaming WIM/ESD split** вҖ” handles install.wim > 4 GB without buffering in RAM
- вң… **No root required** вҖ” uses Android USB Host API + Storage Access Framework
- вң… **Dual boot** вҖ” UEFI 64-bit + Legacy BIOS MBR automatically
- вң… **32-bit UEFI** вҖ” bootia32.efi for Bay Trail tablets (root only)
- вң… **Auto-launch on USB plug** вҖ” opens when you connect OTG drive
- вң… **Live progress log** вҖ” see exactly what's happening during flash
- вң… **Safe USB detection** вҖ” uses `/sys/block/removable` to avoid writing to internal storage

## рҹҺҜ How to Use

```
Step 1 в”Җв”Җв–ә Step 2 в”Җв”Җв–ә Step 3 в”Җв”Җв–ә Step 4 в”Җв”Җв–ә Step 5
 Get ISO    Plug USB   Open App   Configure   Flash!
```

1. **Get a Windows ISO** вҖ” download from Microsoft or use an existing one
2. **Plug in USB OTG drive** вҖ” app opens automatically
3. Tap **Select ISO** вҶ’ pick your `.iso` file
4. Tap **Select USB** вҶ’ pick the OTG drive
5. Pick **Windows Version** or leave on Auto-Detect
6. Configure tweaks вҖ” defaults work great, BypassNRO is on by default for Win 10/11
7. Tap **Flash Windows to USB**
8. Plug USB into your PC вҶ’ boot вҶ’ Windows installs automatically!

## рҹ”§ How It Works

### ISO 9660 Parser

```
ISO file
  в””в”Җв”Җ Sector 16 в”Җв–ә Primary Volume Descriptor
        в””в”Җв”Җ Root directory record
              в”ңв”Җв”Җ /sources/install.wim  вҶҗ split if > 4 GB (streamed, no RAM buffering)
              в”ңв”Җв”Җ /EFI/BOOT/bootx64.efi
              в”ңв”Җв”Җ /bootmgr
              в””в”Җв”Җ ... (all files, FDs closed after each)
```

### Answer File Generation

```
Windows XP  вҶ’  winnt.sif              (root of USB)
Windows 7+  вҶ’  autounattend.xml       (root of USB)
Windows 7+  вҶ’  sources/ei.cfg         (edition override вҖ” now actually written)
```

### Online Account Bypass

```
windowsPE pass
  в””в”Җв”Җ RunSynchronousCommand
        в”ңв”Җв”Җ reg add LabConfig /v BypassNRO      /d 1
        в””в”Җв”Җ reg add LabConfig /v BypassNROCheck /d 1

oobeSystem pass
  в””в”Җв”Җ NetworkLocation = Home
```

### Hybrid Boot

```
USB Drive
в”ңв”Җв”Җ EFI/BOOT/bootx64.efi   вҶҗ UEFI 64-bit  (Level 1 вҖ” always, no root)
в”ңв”Җв”Җ EFI/BOOT/bootia32.efi  вҶҗ UEFI 32-bit  (Level 3 вҖ” root only)
в”ңв”Җв”Җ bootmgr                вҶҗ Legacy BIOS   (Level 1 вҖ” always)
в””в”Җв”Җ Boot/BCD               вҶҗ Boot store
    MBR boot sector        вҶҗ Written via dd bs=512 (Level 2 вҖ” root only)
```
